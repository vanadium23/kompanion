# Coding Conventions

**Analysis Date:** 2026-03-21

## Naming Patterns

**Packages:**
- Lowercase, single word: `auth`, `sync`, `storage`, `library`, `stats`
- Package names match directory names

**Files:**
- Lowercase with underscores: `book_postgres.go`, `progress_test.go`, `interfaces.go`
- Test files: `*_test.go` suffix (e.g., `auth_test.go`)
- Mock files: `mocks_test.go` (generated by mockgen)

**Types (structs, interfaces):**
- PascalCase for exported: `AuthService`, `BookShelf`, `ProgressRepo`
- camelCase for unexported: `response` (in `internal/controller/http/v1/error.go`)
- Interface names end with verb or noun: `Progress`, `Shelf`, `Storage`, `UserRepo`
- Repository interfaces: suffix `Repo` (e.g., `BookRepo`, `UserRepo`)

**Functions/Methods:**
- PascalCase for exported: `NewBookShelf`, `RegisterUser`, `CheckPassword`
- camelCase for unexported: `hashPassword`, `comparePasswords`, `readPrefixedEnv`
- Constructor pattern: `New[Type]` (e.g., `NewConfig`, `NewBookShelf`, `NewMemoryUserRepo`)
- Factory functions in packages: `Init[Service]` (e.g., `InitAuthService`)

**Variables:**
- camelCase: `bookID`, `sessionKey`, `hashedPassword`
- Short names in small scopes: `l` for logger, `pg` for postgres
- Error variables: `Err[Name]` (e.g., `ErrBookAlreadyExists`, `ErrAuth`)

**Constants:**
- PascalCase or UPPER_SNAKE_CASE: `IncorrectPassword`, `UserNotFound`

## Code Style

**Formatting:**
- Standard `gofmt` formatting
- Imports grouped: standard library, external packages, internal packages

**Linting:**
- golangci-lint (run via `make linter-golangci`)

**Comments:**
- Package comments start with "Package [name]:"
- Exported functions have brief comments: `// New -.` style
- TODO comments format: `// TODO: description`

## Import Organization

**Order:**
1. Standard library (context, fmt, os, net, etc.)
2. External packages (github.com/gin-gonic/gin, etc.)
3. Internal packages (github.com/vanadium23/kompanion/internal/...)

**Import Aliases:**
- `pgx "github.com/jackc/pgx/v5"` for pgx driver
- `v1 "github.com/vanadium23/kompanion/internal/controller/http/v1"` for versioned packages

## Error Handling

**Patterns:**

Error definitions at package level in `interface.go` or dedicated file:
```go
var ErrBookAlreadyExists = errors.New("Book already exists")
var IncorrectPassword = errors.New("incorrect password")
```

Error wrapping with context using `fmt.Errorf`:
```go
return entity.Book{}, fmt.Errorf("BookShelf - StoreBook - PartialMD5: %w", err)
```

Error checking with direct comparison:
```go
if err == nil {
    return foundBook, entity.ErrBookAlreadyExists
}
```

Sentinel errors for domain-specific failures:
```go
// In internal/auth/interface.go
var ErrAuth = errors.New("auth error")
var IncorrectPassword = errors.New("incorrect password")
var UserAlreadyCreated = errors.New("user already created")
var UserNotFound = errors.New("user not found")
var DeviceNotFound = errors.New("device not found")
```

## Logging

**Framework:** zerolog via `pkg/logger/logger.go`

**Interface:**
```go
type Interface interface {
    Debug(message interface{}, args ...interface{})
    Info(message string, args ...interface{})
    Warn(message string, args ...interface{})
    Error(message interface{}, args ...interface{})
    Fatal(message interface{}, args ...interface{})
}
```

**Usage pattern:**
```go
l.Info("BookShelf - StoreBook - documentID: %s", koreaderPartialMD5)
l.Error("BookShelf - StoreBook - writeCover: %s", err)
l.Fatal(fmt.Errorf("app - Run - postgres.New: %w", err))
```

**Logger passed to constructors:**
```go
func NewBookShelf(storage storage.Storage, repo BookRepo, l logger.Interface) *BookShelf
```

## Comments

**Package Comments:**
- Start with "Package [name]" followed by brief description
- Example: `// Package app configures and runs application.`

**Function Comments:**
- Brief one-liners with dash suffix: `// New -.`
- Explain purpose, not implementation

**TODO Comments:**
- Format: `// TODO: description`
- Found in code for known improvements (e.g., `// TODO: move session key to separate type`)

## Function Design

**Size:** Functions typically 10-30 lines; complex logic extracted to helpers

**Parameters:** Accept interfaces, not concrete types:
```go
func NewBookShelf(storage storage.Storage, repo BookRepo, l logger.Interface) *BookShelf
```

**Return Values:**
- Return entity structs by value: `entity.Book`
- Return errors as last return value
- Use named returns sparingly

**Constructor Pattern:**
```go
func NewBookShelf(storage storage.Storage, repo BookRepo, l logger.Interface) *BookShelf {
    return &BookShelf{
        storage: storage,
        repo:    repo,
        logger:  l,
    }
}
```

## Module Design

**Exports:**
- Public interfaces defined in `interface.go` or `interfaces.go`
- Implementations unexported, accessed via interface

**Interface-Implementation Separation:**
- Interface: `internal/auth/interface.go` defines `AuthInterface`
- Implementation: `internal/auth/auth.go` implements `AuthService`

**Barrel Files:** Not used; import directly from packages

**Repository Pattern:**
- Repository interfaces define data access: `BookRepo`, `UserRepo`, `ProgressRepo`
- Multiple implementations: memory (for testing), postgres (production)

## Configuration

**Environment Variables:**
- Prefix: `KOMPANION_` (e.g., `KOMPANION_PG_URL`, `KOMPANION_AUTH_USERNAME`)
- Read via `readPrefixedEnv()` helper

**Config Structure:**
```go
type Config struct {
    App
    Auth
    HTTP
    Log
    PG
    BookStorage
}
```

## HTTP Handlers

**Router Pattern:**
- Each route group in separate file: `users.go`, `sync.go`
- Constructor returns routes struct: `newSyncRoutes(handler *gin.RouterGroup, ...)`

**Response Pattern:**
```go
// Success
c.AsciiJSON(http.StatusOK, doc)

// Error with code
c.AsciiJSON(http.StatusBadRequest, gin.H{"message": "Bad request", "code": 4000})
```

**Middleware Pattern:**
```go
func authDeviceMiddleware(a auth.AuthInterface, l logger.Interface) gin.HandlerFunc {
    return func(c *gin.Context) {
        // ... authentication logic
        c.Set("device_name", deviceName)
        c.Next()
    }
}
```

## Dependency Injection

**Via Constructor:**
```go
func Run(cfg *config.Config) {
    // Create dependencies
    pg, _ := postgres.New(cfg.PG.URL, postgres.MaxPoolSize(cfg.PG.PoolMax))
    bookStorage, _ := storage.NewStorage(cfg.BookStorage.Type, cfg.BookStorage.Path, pg)
    authService := auth.InitAuthService(repo, cfg.Auth.Username, cfg.Auth.Password)

    // Inject into handlers
    shelf := library.NewBookShelf(bookStorage, library.NewBookDatabaseRepo(pg), l)
}
```
