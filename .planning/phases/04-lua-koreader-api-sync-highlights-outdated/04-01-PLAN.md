```go
type highlightSyncRequest struct {
    Document   string            `json:"document" binding:"required"`
    Title      string            `json:"title"`
    Author     string            `json:"author"`
    Highlights []entity.Highlight `json:"highlights" binding:"required"`
}

// Each Highlight in the array:
type Highlight struct {
    Text      string `json:"text" binding:"required"`
    Note      string `json:"note"`
    Page      string `json:"page"`
    Chapter   string `json:"chapter"`
    Timestamp int64  `json:"time"`
    Drawer    string `json:"drawer"`
    Color     string `json:"color"`
}
```

Create the `target.lua` file implementing KompanionExporter that inherits from BaseExporter. This is the core exporter logic.

Requirements:
- Inherit from BaseExporter with `name = "kompanion"` and `is_remote = true`
- Implement `isReadyToExport()` - returns true when url, device_name, and device_password are configured
- Implement `getMenuTable()` - returns menu with Setup dialog, Enable toggle, and Help
- Implement `showSetupDialog()` - MultiInputDialog with URL, Device Name, Device Password fields
- Implement `createRequestBody(booknotes)` - transforms booknotes to Kompanion API format
- Implement `export(t)` - iterates booknotes, makes POST to `/syncs/highlights` with Basic Auth

Key implementation details:
- URL field hint: `"http://192.168.1.100:8080"`
- Device Name hint: `"Name from Kompanion Devices page"`
- Device Password field: `text_type = "password"`
- Auth header: `mime.b64(self.settings.device_name .. ":" .. self.settings.device_password)`
- Get document hash: `doc_settings:readSetting("partial_md5_checksum")` from DocSettings
- Request body format matches Kompanion API: `{document, title, author, highlights[]}`
- Each highlight: `{text, note, page, chapter, time, drawer, color}`
- Use `self:makeJsonRequest(url, "POST", body, headers)` for HTTP
- On error: show InfoMessage with error text and return false
- On success: return true
