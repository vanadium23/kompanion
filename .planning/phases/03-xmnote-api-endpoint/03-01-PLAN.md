  <behavior>
    - Test 1: FormatHighlights with single highlight produces correct markdown with title, chapter, page/time, and text
    - Test 2: FormatHighlights groups highlights by chapter correctly
    - Test 3: FormatTitle produces "{author} - {title}" format with newlines replaced by commas
    - Test 4: HashToInt produces stable integer from MD5 hash string
  </behavior>
  <action>
    Create `internal/notes/formatter.go` with the following exported functions:

    1. `FormatHighlights(title, author string, highlights []entity.Highlight) string`
       - Header: "# {title}\n"
       - Author line: "##### {author with newlines replaced by ', '}\n\n"
       - Group highlights by chapter (empty chapter = no chapter heading)
       - For each highlight:
         - Chapter heading if changed: "## {chapter}\n"
         - Page/time line: "### Page {page} @ {timestamp}\n"
         - Timestamp format: "02 January 2006 03:04:05 PM" (matches KOReader's os.date format)
         - Highlight text: "> {text}\n" (per D-07: blockquote format for highlight text)
         - Note if present: "\n---\n{note}\n" (per D-07: plain text for user note)
         - Blank line between highlights: "\n"

    2. `FormatTitle(author, title string) string`
       - Format: "{author} - {title}"
       - Replace "\n" in author with ", " (multi-author handling)
       - This is the title KOReader uses for update detection

    3. `HashToInt(hash string) int`
       - Use CRC32 IEEE checksum to convert string to stable int32
       - Required because Nextcloud Notes API expects integer IDs
       - Import: "hash/crc32"
       - Implementation: `int(crc32.ChecksumIEEE([]byte(hash)))`

    Package name: `notes`
    Import paths needed:
    - "github.com/vanadium23/kompanion/internal/entity"
    - "hash/crc32"
    - "strings"
    - "time"
  </action>
  <verify>

  </verify>
  <done>
    - internal/notes/formatter.go exists with FormatHighlights, FormatTitle, HashToInt functions
    - All functions exported and documented
    - Timestamp format matches "02 January 2006 03:04:05 PM"

  </done>
