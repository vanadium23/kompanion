# Phase 2 -- UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Go server-side rendering with goview) |
| Preset | not applicable |
| Component library | none (custom CSS with semantic HTML) |
| Icon library | none |
| Font | system default (inherited from master layout) |

**Note:** This project uses Go with goview templates and custom CSS. No component library is used. Design follows existing patterns in `web/templates/` and `web/static/static.css`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

**Evidence from codebase:**
- `book-card`: `gap: 1rem` (16px), `padding: 1rem` (16px), `margin-bottom: 1.5rem` (24px)
- `edit-book-article`: `gap: 20px`, `padding: 20px`
- Reading stats section: no explicit margin, follows flow

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | CSS var(--background) | Page background |
| Secondary (30%) | CSS var(--card-background) | Cards, sections |
| Accent (10%) | CSS var(--text-color) | Highlight borders, emphasis |
| Destructive | CSS var(--text-color-alt) | Muted secondary text |

**CSS Variables in use:**
- `--text-color`: Primary text and borders
- `--text-color-alt`: Secondary/muted text
- `--border-thickness`: Border width

Accent reserved for: highlight card left border, reading stats section heading

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | N/A (read-only display, no CTA) |
| Empty state heading | Highlights |
| Empty state body | No highlights synced yet. Use KOReader to highlight text in this book. |

**Content structure for highlight cards:**
- Highlight text (required): `<blockquote class="highlight-text">{{ .Text }}</blockquote>`
- Note (optional): `<div class="highlight-note"><strong>Note:</strong> {{ . }}</div>`
- Meta (optional): Page and chapter information
- Section heading: "Highlights" with count subtitle

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| View book details | Highlights section appears below reading stats (if any highlights exist) |
| No highlights | Empty state message displays with helpful instruction |
| Server error fetching highlights | Fails silently with empty slice, logs error |

**Read-only enforcement:**
- No form elements in highlights section
- No edit/delete buttons
- No input fields
- Display-only HTML structure (blockquote, div, footer)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Section heading | Use `<hgroup>` with `<h3>` for "Highlights" |
| Semantic structure | Use `<section>`, `<article>`, `<blockquote>` |
| Empty state | Informative message explains why no highlights |
| Color contrast | Use existing CSS variables (already themed) |
